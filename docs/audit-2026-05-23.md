## 🟡 IMPORTANT — should be addressed before next public iteration

### 7. Missing `useSEO()` on internal surfaces

MarketFlow role surfaces and the offer studio render without page titles or meta
descriptions. Even though they're behind auth, they are indexable if a session
leaks. Recommended: add a minimal `useSEO({ title, description, noIndex: true })`
to each:

- `client/src/pages/marketflow-dashboard.tsx`
- `client/src/pages/marketflow-deals.tsx`
- `client/src/pages/marketflow-negotiate.tsx`
- `client/src/pages/marketflow-submit.tsx`
- `client/src/pages/offer-studio.tsx`
- `client/src/pages/my-deals.tsx`
- `client/src/pages/dealflow-community.tsx`
- `client/src/pages/dealflow-messages.tsx`
- `client/src/pages/admin-strategy-lab.tsx`
- `client/src/pages/admin-vendors.tsx`

### 8. Accessibility — icon-only buttons + decorative imgs

- `client/src/components/footer.tsx:84,190` — `alt=""` on decorative images is correct but pair with `aria-hidden="true"` so screen readers skip the empty alt entirely.
- `client/src/components/navigation.tsx:357` — logo `<img>` should have `aria-hidden="true"` since the `<a>` already carries `aria-label`.
- Any icon-only buttons rendered inside the unmounted home sections — moot if §5 cleanup happens.

### 9. Strategy Lab page — hex literals instead of tokens

`client/src/pages/strategy-lab.tsx` was carried forward from pre-Foundation
Reset and contains:
- Line 1552: gradient `from-[#E8DBC5] via-[#D4B483] to-[#C17A4A]` — out-of-palette colors.
- Lines 1568, 2191, 2235, 2280, 2351, 2478, 2562: `bg-[hsl(var(--copper))] text-white` on CTAs. Should be `bg-primary text-primary-foreground` to inherit dark-mode handling.

Note: Task #126 just merged a rebuild. If the rebuild **replaced** strategy-lab.tsx these findings are moot — verify before fixing. If it merely **added new components** alongside it, these CTAs are still live.

### 10. Voice & locked phrases — verified clean on mounted surface

Public-voice test passes (276/276). All required locked phrases present on
home, about, footer. No forbidden phrases on the v1 mounted public pages.
No spaced em-dashes outside the documented exclusions.

### 11. Dark-mode token discipline

`development.tsx` lines 73, 86, 95 use `text-white` literals. These sections
render on a dark hero photo so they read correctly today, but the literal
should be `text-cream` (matches the rest of the dark-section convention).
Low-risk fix, recommend bundling into the next visual polish pass.
