MPORTANT: MarketFlow must support TWO distinct deal categories with different forms and workflows:

1) Wholesale DealFlow (Assignment Marketplace)

Who posts: Wholesalers (verified)
Goal: Find an assignee/cash buyer/operator to take the contract so wholesaler earns an assignment fee.
This is NOT capital raising.

Wholesale Submission Form (Form A)

Fields (MVP):

Property address (or masked option)

Contract price

Assignment fee asked

Close date

ARV (optional)

Rehab estimate (optional)

Negotiation allowed on assignment fee? (yes/no)

JV allowed with another wholesaler? (yes/no)

Wholesale Deal Card UI

Show:

Contract price

Assignment fee

Close date

Negotiable badge (if allowed)
Buttons:

Request Assignment / Make Offer

Counter Assignment Fee (only if negotiable)

Request JV (wholesaler-to-wholesaler) (only if allowed)

JV Flow: A wholesaler can request to JV because they have a buyer; they propose assignment fee split; accept/counter/reject.

2) Capital Raise DealFlow (Operator Project Raising Capital)

Who posts: Dreamscaper/Operator (builder/developer/flipper)
Goal: Raise capital for a project. Investors can accept terms or counteroffer terms.

Capital Raise Submission Form (Form B)

Fields (MVP):

Project name + address (or masked)

Strategy (flip/rental/new construction)

Total project budget (optional placeholder)

Capital raise target amount (required)

Minimum investment (optional)

Operator proposed terms summary (required): debt/equity/JV structure + expected return structure (educational only)

Timeline estimate

Capital Raise Deal Card UI

Show:

Capital target

Amount committed (mock data ok)

Progress bar: committed vs target
Buttons:

Accept Terms

Counter Terms (routes to negotiation room)

Commit Capital (placeholder / under construction)

Message Operator

Capital Raise Negotiation Room (Different from wholesale)

Negotiation focuses on financing/partnership terms:

investment amount

structure type (debt/equity/JV)

preferred return / interest

profit split

maturity / timeline

notes/contingencies
Actions: Submit / Accept / Reject / Counter

When committed reaches target: status becomes “Funded”.

Marketplace UX Requirement

On the Deals page, add tabs or filters:

Wholesale Assignments

Capital Raises
Each tab uses its own cards, actions, and submission form.

If anything is not implemented, show it clearly as “Under Construction” without breaking the UX.
