======================================================
0. CONTEXT & GLOBAL PRINCIPLES
======================================================

High-level architecture:

- This app = **Public Website + Marketplace**
- HQ app = **Internal OS** (already being built separately)
- Both share **Supabase** (auth, tables, storage, policies)
- This app should:
  - Attract & educate visitors
  - Collect leads (distressed sellers + wholesalers)
  - Run a marketplace for deals & capital
  - Provide “mini-office” dashboards for external users
  - Sync cleanly with HQ via shared tables

Key user types (external):

- Distressed property sellers (homeowners)
- Wholesalers (with contracts, or bird-dogs)
- Dreamscapers (flippers/developers using the marketplace)
- Investors (providing capital)
- Buyers (retail + investment buyers)
- Admin (Pegasus internal staff managing the website/marketplace)

**IMPORTANT**:  
There are **TWO “seller” types**:
1) **Distressed property owner** (no contract, just “sell my house”)  
2) **Wholesaler** (has a contract or access to a deal, wants to submit it)

These must be treated as two distinct flows and stored distinctly.

**You must implement:**

5. PAGE-BY-PAGE SPECIFICATION (PUBLIC WEBSITE)
======================================================

### 5.1 `/` Home

Goals:
- Introduce Pegasus DreamScapes (brand, mission, “Where Designed Profits Are Crafted, and Communities Are Elevated.”)
- Explain the core pillars:
  - Sell your property
  - Invest with Pegasus
  - Buy a Pegasus home
  - Partnerships and Marketplace (wholesalers & dreamscapers)

Content sections:
- Hero with CTA buttons:
  - “Sell My Property”
  - “Invest With Pegasus”
  - “Explore Marketplace”
- How It Works (for sellers, investors, buyers)
- Short explanation of Dreamscapers (what a Dreamscaper is)
- Trust elements (track record placeholders, testimonials, logos, etc.)
- “Get Started” CTA linking to /sell, /invest, /buy, /marketplace as appropriate.

### 5.2 `/sell` (Distressed sellers)

This page is ONLY for property owners who want to sell (no contract, no wholesaling).

Include:
- Clear messaging: “Sell your house as-is”, “Fast offers”, “No repairs / No fees” etc.
- Explain the process in 3–5 simple steps.
- A single, clean form:

Form fields (mapped to `seller_leads`):
- Name
- Phone
- Email
- Property address (with Google Places autocomplete if possible)
- Property type (single-family, duplex, etc.)
- Reason for selling (list; include “motivation” concept)
- Timeline to sell (ASAP, 30–60 days, 3+ months)
- Property condition (basic scale)
- Notes

On submit:
- Insert into `seller_leads` with:
  - `source = 'Website - Sell'`
  - `status = 'New'`
- Show a success message or redirect to a thank-you page.
- Do NOT require login for distressed sellers.

### 5.3 `/submit-deal` (Wholesalers / contracts)

This is **not** for distressed homeowners; it’s for wholesalers and Pegasus acquisitions.

Behavior:
- If not logged in:
  - Show explanation of program for wholesalers.
  - CTA → login/sign-up as Wholesaler.
- If logged in with role `wholesaler` or `pegasus_wholesaler`:
  - Show detailed deal submission form.

Form fields (mapped to `wholesale_deals`):
- Deal title
- Property address (Places autocomplete if possible)
- Property type
- ARV estimate
- Asking price
- Estimated repairs
- Assignment fee
- Photos (URLs or file upload stub)
- Occupancy status
- Timeline to close
- “Do you have the contract signed?” (yes/no)
- Notes / Deal Story

On submit:
- Insert into `wholesale_deals` with:
  - `wholesaler_id = current user`
  - `status = 'Under Review'`
  - `is_public = false`
- Show success, and make it visible in their Wholesaler Dashboard.

### 5.4 `/invest`

For investors providing capital.

Content:
- Explain Pegasus model:
  - How investor capital is used
  - Types of returns (debt vs equity)
  - Typical timelines
  - Risk & transparency
  - Role of Dreamscaper

Include:
- Examples of potential deals (generic)
- Explanation of the Pegasus Marketplace for investors (they can browse opportunities)

CTAs:
- “Create Investor Account” → pre-select investor role on signup
- “Log in as Investor”

Later, the investor’s dashboard will show real deals from `marketplace_deals`.

### 5.5 `/buy`

For buyers of:
- Retail Pegasus finished homes
- Investment/wholesale deals

Split the page into 2 sections:

1) **Buy a Finished Pegasus Home**
   - Explanation: renovated, staged, quality work.
   - CTA: “See Finished Homes (Retail Buyer)”
   - On click:
     - Guide user to login/signup with `buyer_retail` role.

2) **Buy Investment / Wholesale Deals**
   - Explanation: investor-focused properties, wholesale spreads.
   - CTA: “See Investment Deals (Buyer/Investor)”
   - On click:
     - Guide user to login/signup with `buyer_investment` role.

Those roles then control what their Dashboard shows.

### 5.6 `/marketplace` (Landing page)

This is the public explainer for the Pegasus Marketplace.

Content:
- Explain how wholesalers, dreamscapers, and investors connect.
- Show the pipeline:
  1) Wholesaler submits deals
  2) Dreamscaper evaluates and runs the project
  3) Investors fund selected deals
  4) Pegasus supports the ecosystem
- Have distinct CTAs:
  - “I’m a Wholesaler” → login/signup as wholesaler
  - “I’m a Dreamscaper (Flipper/Developer)” → login/signup as dreamscaper
  - “I’m an Investor” → login/signup as investor
  - “I’m a Buyer” → buyer signup options

Do not show actual deals on this page if user is not logged in; just show teaser or “sign in to view marketplace deals.”

6. MARKETPLACE DASHBOARDS (MINI-OFFICES)
======================================================

Each role must have a dashboard that feels like a small “office.”

Shared elements:

- Top-level summary cards (counts, stats)
- Tables or cards of relevant items
- Quick links to primary actions
- Access to profile & settings

### 6.1 `/marketplace/wholesaler` (Wholesaler Dashboard)

Audience:
- External wholesale users
- Pegasus internal acquisitions (pegasus_wholesaler) will use HQ heavily, but can still see their deals here.

Sections:

1) Summary:
   - Total submitted deals
   - Under review
   - Approved / Listed
   - Rejected

2) “My Deals”
   - Table or cards for all `wholesale_deals` where `wholesaler_id = current user`.
   - Columns: title, address, status, created_at, last_update.
   - Filter by status.

3) “Submit New Deal” button:
   - Routes to `/submit-deal` for wholesaler.

Later integration:
- Show interest from dreamscapers (placeholder field).
- Show if a deal has been converted to `marketplace_deals`.

### 6.2 `/marketplace/dreamscaper` (Dreamscaper Dashboard)

Audience:
- Flippers/developers using Pegasus marketplace, external + internal Dreamscapers.

Sections:

1) Summary:
   - Deals available to review
   - Deals they are working on
   - Deals currently raising capital
   - Completed deals

2) “Browse Wholesale Deals”
   - List of deals from `wholesale_deals` with `status = 'Approved'` and `is_public = true`.
   - For each card:
     - Address, ARV, asking, repairs.
     - Wholesaler name.
     - Badges (Pegasus Wholesaler vs external).
     - Basic trust info from `user_reputation`.

3) “My Deals”
   - Deals in `marketplace_deals` where `dreamscaper_id = current user`.
   - Show columns:
     - Deal name, status (Draft, Raising Capital, Funded, Completed)
     - Capital needed vs raised, timeline.

Actions (can be stubbed in UI, logic later):
- “Create Marketplace Deal” from a wholesale deal.
- “Start Raising Capital” checkbox/flow for a project.
- “Mark as Completed.”

### 6.3 `/marketplace/investor` (Investor Dashboard)

Audience:
- Investors, capital partners.

Sections:

1) Summary:
   - Active commitments
   - Total capital committed
   - Active projects
   - Completed investments

2) “Available Deals”
   - List of `marketplace_deals` with `status = 'Raising Capital'`.
   - Show:
     - Address / project name
     - Dreamscaper info + Pegasus badge
     - Strategy (Flip, BRRRR, etc.)
     - Target returns & timeline
     - Capital needed vs raised (progress bar)
   - CTAs:
     - “View Details”
     - “Express Interest” → insert row into `investor_interests`.

3) “My Investments”
   - Deals where `investor_id = current user` in `investor_interests`.
   - Show statuses and simple metrics.

### 6.4 `/marketplace/buyer` (Buyer Dashboard)

Audience:
- Retail and investment buyers.

Behavior:
- If role = `buyer_retail`:
  - Show “Saved Homes”, “Inquiries/Tours”, “Offers Submitted” (basic placeholders).
- If role = `buyer_investment`:
  - Show “Saved Deals”, “Offers/LOIs”, “Closed Deals” (basic placeholders).

Use `buyer_saved_items` as a generic saved/favorites table.

### 6.5 `/marketplace/admin` (Marketplace Admin)

This is NOT HQ. It is a **lightweight admin view** for this app only, for internal staff to:

- Review and approve wholesale deals (`wholesale_deals`).
- Toggle `is_public` and `status`.
- Inspect user profiles.
- Manage basic content flags (e.g., mark deals as featured).

Make it simple and safe; deeper internal operations live in HQ.
